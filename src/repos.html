Repository Types

The first part of the repository name is always the repository type except for Utility Repository. For example, if the application name is called Learning Java Script then the repository name would be software-learning-js . A banner should appear in the Read Me file so the church member can easily identify what type of repository. Having banner design by members our church recognizes distinguish members of our congregation. To nominate a distinguished scientist for banner consideration please submit require to the Church of Scyence Banner Committee.

Charles Darwin - was an English naturalist, geologist, and biologist, widely known for his contributions to evolutionary biology.

Utility Repository - it contains Photographs, the Church Website, Holy Documents, and anything from the church to use but there is no code base in these repositories.

Isaac Newton - was an English polymath active as a mathematician, physicist, astronomer, alchemist, theologian, and author who was described in his time as a natural philosophy.

Software Repository - it contains code that can be compiled as such as a Java Application or interpreted as such as an Html Application which is rendered by the browser.

Galileo Galilei - Galileo has been called the father of observational astronomy, modern-era classical physics, the scientific method, and modern science.

Tutorial Repository - This repository only has items such as Jupyter Notebooks and documentation. This repository has reference material. There is no code base in these repositories.

Thomas Edison - He developed many devices in fields such as electric power generation, mass communication, sound recording, and motion pictures.

Sandbox Repository - This repository does not contain any code, only reference material. The various items in this repository are documents, Jupyter Notebook, and diagrams.

Marie Curie - was a Polish and naturalized-French physicist and chemist who conducted pioneering research on radioactivity.

Stater Repository -This repository contains the initial setup code before the developer adds software specifications code. The readme file should include a list of screenshots and libraries.
